🔌 Connectors app/connectors/

  • base.py — BaseConnector, RawPage, RawDocument
  • pdf_connector.py — pdfplumber, 1 page = 1 RawPage
  • markdown_connector.py — H1/H2 heading split
  • notion_connector.py — Notion API, pagination

📇 Indexing app/indexing/

  • tokeniser.py — text→tokens, stopwords, TF
  • bm25.py — pure BM25 math, no I/O
  • index_writer.py — RawDoc→SQLite, only DB writer

☁️ PageIndex app/pageindex/

  • client.py — submit, poll, retrieve, chat, delete
  • Only file that calls pageindex.ai API
  • PINode, PIRelevantContent dataclasses

🔍 Retrieval app/retrieval/

  • bm25_retriever.py — query→BM25Hits via SQLite
  • pageindex_retriever.py — query→PINodes via API
  • hybrid_retriever.py — RRF fusion, alpha-weighted

🗄️ Database app/db/

  • models.py — documents, pages, page_index, query_log
  • document_repo.py — CRUD for docs + pages
  • index_repo.py — BM25 postings read/write
  • query_log_repo.py — audit trail writer

🤖 Generation app/generation/

  • llm_client.py — Anthropic + OpenAI behind one API
  • prompt_builder.py — assembles ranked context
  • citation_formatter.py — wraps answer + sources

🌐 API Routes app/api/routes/

  • ingest.py — POST /ingest/upload, /ingest/notion
  • query.py — POST /query (full pipeline)
  • documents.py — GET/DELETE /documents, /health

🐳 Deployment

  • Dockerfile — multi-stage production image
  • docker-compose.yml — prod stack + nginx
  • docker-compose.dev.yml — hot-reload dev
  • deploy/k8s/ — K8s manifests + HPA
  • .github/workflows/ — CI/CD pipelines
# Step Input Output Layer Storage
1 File received multipart/form-data bytes raw bytes + filename API memory
2 Connector raw bytes RawDocument (title, pages[]) Connector memory
3a BM25 index RawDocument Page rows + PageIndexEntry rows IndexWriter SQLite
3b PageIndex submit PDF bytes pi_doc_id string PageIndex API cloud
4b PageIndex poll pi_doc_id status=completed + tree PageIndex API cloud → SQLite
5 Query received QueryRequest JSON query string + options API —
6a BM25 retrieve query string BM25Hit[] (page_id, score, terms) BM25Retriever SQLite read
6b PageIndex retrieve query + pi_doc_ids[] PINode[] (title, page_index, content) PIRetriever cloud API
7 RRF Fusion BM25Hits + PINodes HybridHit[] ranked by RRF score HybridRetriever memory
8 Page hydration HybridHit page_ids[] PageContext[] with full text PageRepository SQLite read
9 Prompt build PageContext[] + query (system_prompt, user_message) PromptBuilder memory
10 LLM call system + user prompts LLMResponse (content, tokens) LLMClient LLM API
11 Citation format LLMResponse + PageContexts AnswerResult (answer, sources[]) CitationFormatter memory
12 Audit log query + result + latency QueryLog row QueryLogRepo SQLite write
13 Response AnswerResult QueryResponse JSON API —

⚙️ BM25 Tuning

VariableDefaultDescription
BM25_K11.5Term frequency saturation. Higher = more weight to repeated terms
BM25_B0.75Length normalisation. 0=none, 1=full normalisation
BM25_TOP_K10BM25 candidates before RRF fusion

🌲 PageIndex Tuning

VariableDefaultDescription
PI_TOP_K5PageIndex nodes returned before fusion
PI_POLL_INTERVAL_SEC2Seconds between status polls
PI_POLL_TIMEOUT_SEC120Max wait for PageIndex processing

⚖️ Hybrid Fusion

VariableDefaultDescription
HYBRID_ALPHA0.50.0 = pure BM25 · 1.0 = pure PageIndex · 0.5 = equal
FINAL_TOP_K5Pages sent to LLM after fusion
